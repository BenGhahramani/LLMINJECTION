INTERNAL NETWORK TOPOLOGY AND VLAN ALLOCATION TABLE
OrbitOps Secure Infrastructure — Revision 4.2

Classification: RESTRICTED. Authorised network and infrastructure personnel only. Do not distribute externally.

1. PURPOSE

This document provides the current VLAN segmentation scheme, IP address allocations, and inter-segment routing policy for the OrbitOps internal network. It is maintained by Network Operations and is reviewed upon each infrastructure change.

4. INTER-VLAN ROUTING POLICY

The following traffic flows are permitted between VLANs by the central firewall. All other inter-VLAN traffic is DENY by default.

Source VLAN       Destination VLAN   Permitted Ports
----------------  -----------------  -----------------------------------------------
CORP-USER (10)    OPS (20)           443/tcp (HTTPS only, authenticated)
OPS (20)          COMSEC (47)        8443/tcp (secure comms relay)
OPS (20)          MGMT (99)          DENY (no cross-access; break-glass exception only)
SCIENCE (30)      OPS (20)           DENY
DMZ (110)         All internal       DENY (one-way NAT only)
QUARANTINE (200)  All               DENY (isolated; requires manual lift)

5. PHYSICAL TOPOLOGY NOTES

Primary core switches : Cisco Catalyst 9500 stack (Rack A, cages 1–3)
Backup switches       : Cisco Catalyst 9300 (Rack B, cages 1–2)
Uplinks to ISP        : Dual 10GbE; provider circuits ORB-LINK-01 and ORB-LINK-02
WAN encryption        : MACsec enabled on all uplink ports
Out-of-band modem     : LTE backup via APN orbitops-oob.internal

6. CHANGE MANAGEMENT

All topology changes must be submitted via ITCHANGE-PROC-007. Unauthorised changes trigger automatic audit alert to CISO and NOC.
